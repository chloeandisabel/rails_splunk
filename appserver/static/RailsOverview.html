Welcome to the Splunk for Ruby on Rails Community App


This app contains a set of field extractions, searches, reports, and dashboards for analyzing Ruby on Rails production log data.


The easiest way to get started is to index some sample data to see how it all works:

  1. Go to the Manager->Data Inputs->Files and Directories page
  2. Select "Monitor a file or directory"
  3. Enter the directory path "$SPLUNK_HOME/etc/apps/rails/logs". There should be two files in this directory: production.sample1.log and production.sample2.log
  4. IMPORTANT: Before adding the input, select a manual sourcetype and enter "rails". This app requires that your data is indexed with sourcetype=rails in order for the included extractions and reports to function correctly.
  5. OPTIONAL: Specify a different index to index the sample data and add it to your default indexes, if you do not want the sample data to pollute your main index.
